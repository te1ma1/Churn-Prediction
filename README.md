# Churn Prediction — Прогнозирование оттока клиентов телеком-компании


Прогнозирование оттока клиентов на основе датасета **Telco Customer Churn** (Kaggle).  
Цель — выявить клиентов с высоким риском ухода и дать бизнес-рекомендации по удержанию.

## Что сделано
- Полный цикл анализа: от загрузки данных до бизнес-инсайтов
- Качественная EDA с визуализациями (matplotlib + seaborn + plotly)
- Предобработка данных и базовое моделирование
- Сохранённые модели и графики
- Чёткая структура проекта (лучшие практики junior DS)

## Запуск
git clone https://github.com/te1ma1/Churn-Prediction.git

cd Churn-Prediction

pip install -r requirements.txt

jupyter notebook
notebook/01_EDA.ipynb → notebook/02_modeling.ipynb

## Ключевые инсайды из EDA 
Основные драйверы оттока:
<ul> 
  <li>Тип контракта (Month-to-month — самый высокий риск)</li>
  <li>Способ оплаты (Electronic check — высокий отток))</li>
  <li>Высокий MonthlyCharges (> $80)</li>
  <li>Первые 3 месяца подписки — критический период</li>
</ul>
Гендер почти не влияет на отток

## Моделирование
<ul>
  <li>Предобработка: Label Encoding категориальных признаков, обработка TotalCharges, удаление строк с tenure == 0</li>
  <li>Разбиение: 70/30 (stratify по Churn)</li>
  <li>Оценка по бизнес-метрикам: accuracy, precision, recall, F1, ROC-AUC</li>
  <li>Сравнение двух моделей по экономической эффективности и охвату удержания клиентов</li>
</ul>
Рекомендация по выбору модели:
<ul>
  <li>Ограниченный бюджет → первая модель (экономически выгоднее)</li>
  <li>Максимальное удержание клиентов → вторая модель</li>
  <li>Гибридный подход + ручная валидация сомнительных случаев</li>
</ul>

## Бизнес-рекомендации
<ul>
  <li>Продвигать долгосрочные контракты (скидки за 1–2 года)</li>
  <li>Стимулировать автоплатежи вместо Electronic check</li>
  <li>Пересмотреть тарифы для клиентов с высокой абонентской платой</li>
  <li>Усилить онбординг в первые 90 дней</li>
</ul>
